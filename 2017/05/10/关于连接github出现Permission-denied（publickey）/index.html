关于连接github出现Permission denied（publickey）

解决推送远程仓库是提示Permission denied（publickey）

问题描述

在连接远程仓库后，使用git push推送时提醒 Permission denied (publickey)

使用ssh git@github.com测试后显示

说明电脑没有连接到远程仓库

之前已经放了一个公钥在GitHub上，显示没有使用过

使用ssh -T -v git@github.com查看连接日志

从下面这段代码发现ssh查找的路径为/Users/apple/.ssh/而我之前使用的公钥是存放在根目录下的

解决方法

  1. 进入.ssh查看文件，可以看到存在id_rsa和id_rsa.pub分别是私钥和公钥。

  2. 打开.ssh下的公钥文件把内容添加到GitHub的SSH keys

    再次验证 ssh -T git@github.com

  3. 验证推送

成功推送到远程仓库
