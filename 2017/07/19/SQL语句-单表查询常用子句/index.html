SQL语句-单表查询常用子句

SQL单表查询常用子句,如select、where、order by、聚集函数等

1. select子句

列名重命名 旧名 AS 新名；AS可以省略

过滤重复元素 在select子句中加上distinct select distinct * from student;

2. where子句

查询条件 谓词
比较 =，>=,<=,>,<,<>
确定范围 between and，not between and
确定集合 in，not in
字符匹配 like，not like
空值 is null，is not null
逻辑运算符 and，or，not

  • 比较大小

    where age > 19;

  • 确定范围

    列名 between···and
    where age between 19 and 20;

  • 确定集合

    列名 in 集合
    where dept in (“‘计算机’,’管理’”);

  • 字符串匹配

    列名 like ‘<匹配串>’ [escape’<换码字符>’] %代表任意长度字符 _代表任意带个字符

    where name like ‘王%’;

  • 空值判断

    where date is null;

  • 逻辑运算

    and or 连接多个查询语句

3. order by 子句

用于对查询结果进行排序，可以按照一个或多个属性列的升序(ASC)或降序(DESC)

SELECT id,address,city,name,price FROM realestate_residential_info_price201707 WHERE city = '东莞市' ORDER BY price;

4. 聚集函数

聚集函数 解释
count([distinct/all] *) 统计元组个数
count([distinct/all] 列名) 统计一列中值的个数
sum([distinct/all] 列名) 计算一列值的总和
avg([distinct/all] 列名) 计算一列值的平均值
max([distinct/all] 列名) 求一列值中的最大值
min([distinct/all] 列名) 求一列值中的最小值

5. group by

分组就是将具有相同特征（在一个或多个列上的值相同）的元组分配到同一个组，作为一个整体进行处理
group by <列名>

select语句中出现了group by 子句后，在select子句中只能中出现分组列的列名以及聚集函数

select dept,sex count(*) from student group by dept,sex order by dept

6. having 子句

where子句与having子句的区别在于作用对象不同。where子句对select-from语句查询结果的元组进行选择，从中选择满足条件的元组。having子句作用于执行select-from-where-group by语句后得到的分组，从中选择满足条件的分组

having <条件表达式>

条件表达式是由分组列、聚集函数和常数构成的有意义的句子。
select cno,count(*) from sc group by no having count(*) >2 and cno like '%7%'

7. 注释

注释以两个减号开始，回车结束
--查询东莞市的POI SELECT id,address,city,name FROM t_base_poi WHERE city = '东莞市' and ntype LIKE '120%';
